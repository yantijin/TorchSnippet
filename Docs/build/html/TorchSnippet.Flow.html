class TorchSnippet.Flow.actnorm.ActNorm(num_features: int, axis: int = -1, event_ndims: int = 1, scale: Union[str, TorchSnippet.typing_.ActNormScaleType] = 'exp', initialized: bool = False, epsilon: float = 1e-05, dtype: str = 'float32')¶

Bases: TorchSnippet.Flow.base.FeatureMappingFlow

ActNorm proposed by (Kingma & Dhariwal, 2018).

y = (x + bias) * scale; log_det = y / scale - bias

bias and scale are initialized such that y will have zero mean and unit variance for the initial mini-batch of x. It can be initialized only through the forward pass. You may need to use BaseFlow.invert() to get a inverted flow if you need to initialize the parameters via the opposite direction.

class TorchSnippet.Flow.base.BaseFlow(x_event_ndims: int, y_event_ndims: int, explicitly_invertible: bool)¶

Bases: TorchSnippet.Layers.base.BaseLayer

Base class for normalizing flows.

A normalizing flow transforms a random variable x into y by an (implicitly) invertible mapping \(y = f(x)\), whose Jaccobian matrix determinant \(\det \frac{\partial f(x)}{\partial x} \neq 0\), thus can derive \(\log p(y)\) from given \(\log p(x)\).

explicitly_invertible: bool = None¶

Whether or not this flow is explicitly invertible?

If a flow is not explicitly invertible, then it only supports to transform x into y, and corresponding \(\log p(x)\) into \(\log p(y)\). It cannot compute \(\log p(y)\) directly without knowing x, nor can it transform x back into y.

forward(input: torch.Tensor, input_log_det: Optional[torch.Tensor] = None, inverse: bool = False, compute_log_det: bool = True) → Tuple[torch.Tensor, Optional[torch.Tensor]]¶

Transform x into y and compute the log-determinant of f at x (if inverse is False); or transform y into x and compute the log-determinant of f^{-1} at y (if inverse is True).

Args:

input: x (if inverse is False) or y (if inverse is True). input_log_det: The log-determinant of the previous layer.

Will add the log-determinant of this layer to input_log_det, to obtain the output log-determinant. If no previous layer, will start from zero log-det.

inverse: See above. compute_log_det: Whether or not to compute the log-determinant?

Returns:

The transformed tensor, and the summed log-determinant of the previous flow layer and this layer.

TorchSnippet.Flow.coupling module¶

class TorchSnippet.Flow.coupling.CouplingLayer(shift_and_pre_scale: torch.nn.modules.module.Module, axis: int = -1, event_ndims: int = 1, scale: Union[str, TorchSnippet.Flow.scale.BaseScale, Type[TorchSnippet.Flow.scale.BaseScale], Callable[[], TorchSnippet.Flow.scale.BaseScale]] = 'exp', secondary: bool = False, sigmoid_scale_bias: float = 2.0, epsilon: float = 1e-05)¶

Bases: TorchSnippet.Flow.base.FeatureMappingFlow

A general implementation of the coupling layer (Dinh et al., 2016).

Basically, a CouplingLayer does the following transformation:

x1, x2 = split(x)
if secondary:
    x1, x2 = x2, x1

y1 = x1

shift, pre_scale = shift_and_scale(x1)
if scale_type == 'exp':
    y2 = (x2 + shift) * exp(pre_scale)
elif scale_type == 'sigmoid':
    y2 = (x2 + shift) * sigmoid(pre_scale + sigmoid_scale_bias)
elif scale_type == 'linear':
    y2 = (x2 + shift) * pre_scale
else:
    y2 = x2 + shift

if secondary:
    y1, y2 = y2, y1
y = tf.concat([y1, y2], axis=axis)

The inverse transformation, and the log-determinants are computed according to the above transformation, respectively.

Bases: TorchSnippet.Flow.base.BaseFlow

A flow which reshapes the last x_event_ndims of x into y_event_shape.

Usage:

# to reshape a conv2d output into dense input
flow = ReshapeFlow(x_event_ndims=3, y_event_shape=[-1])
x = T.random.randn([2, 3, 4, 5])
y, log_det = flow.transform(x)

# y == T.reshape(x, [2, -1])
# log_det == tf.zeros([2])
